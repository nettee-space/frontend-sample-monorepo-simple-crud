```bash
# Windows (cmd)
mkdir apps\server\src\data
type nul > apps\server\src\data\db.json

# Windows (PowerShell)
New-Item -ItemType Directory -Path "apps\server\src\data" -Force
New-Item -ItemType File -Path "apps\server\src\data\db.json" -Force

# Linux/Mac (Bash)
mkdir apps/server/src/data && touch apps/server/src/data/db.json
```

## API 문서

API 문서는 서버 실행 후 다음 URL에서 확인할 수 있습니다
http://localhost:4000/

### Swagger UI 사용법

1. **API 엔드포인트 탐색**

   - 각 API 그룹(Posts, Comments 등)을 클릭하면 세부 엔드포인트들이 표시됩니다
   - 녹색(GET), 파란색(POST), 노란색(PUT), 빨간색(DELETE)으로 HTTP 메서드를 구분합니다

2. **API 테스트 방법**

   - 테스트하고 싶은 엔드포인트를 클릭합니다
   - `Try it out` 버튼을 클릭합니다
   - 필요한 파라미터나 요청 본문을 입력합니다
   - `Execute` 버튼을 클릭하여 API를 실행합니다
   - 응답 결과를 바로 확인할 수 있습니다

3. **Schema 확인**
   - 각 API의 Request/Response 데이터 구조를 Schema에서 확인할 수 있습니다
